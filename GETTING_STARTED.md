## Study Tips

### Phase 1: DSA (Weeks 1-6)
1. Master each data structure:
   - Arrays, Linked Lists, Stacks, Queues
   - Trees, Graphs
   - Hash Maps, Heaps
2. Practice 50-100 LeetCode problems
3. Focus on: Time/Space complexity, Edge cases
4. Code in your preferred language (Java/Python/JS)

### Phase 2: LLD (Weeks 7-12)
1. Learn design patterns thoroughly
2. Understand OOPS principles
3. Practice 5-8 LLD design problems
4. Design: Parking Lot, ATM, Library, URL Shortener, etc.

### Phase 3: HLD (Weeks 13-18)
1. Learn architecture patterns
2. Study database design
3. Understand scalability concepts
4. Design simple systems first

### Phase 4: System Design (Weeks 19-26)
1. Learn core concepts
2. Study real-world case studies
3. Practice 5-7 system design rounds
4. Design: Twitter, Instagram, Uber, Netflix, etc.

### Phase 5: Specialized (Weeks 27+)
1. Dive deeper into AI frameworks
2. Practice behavioral interviews
3. Polish communication skills
4. Take full mock interviews

## Frequently Used Patterns

### Two Pointers
Used in: Arrays, Strings, Linked Lists
Problems: Remove Duplicates, Container with Most Water, Valid Palindrome

### Sliding Window  
Used in: Strings, Arrays, Subarray problems
Problems: Longest Substring, Max Subarray Sum, K-Distinct Elements

### BFS/DFS
Used in: Trees, Graphs, Matrices
Problems: Binary Tree Traversal, Connected Components, Shortest Path

### Binary Search
Used in: Sorted Arrays, Rotated Arrays
Problems: Search Target, First/Last Position, Find Peak

### Greedy
Used in: Arrays, Graphs, DP
Problems: Jump Game, Activity Selection, Huffman Coding

### Dynamic Programming
Used in: Subsequences, Shortest Path, Optimization
Problems: Fibonacci, Coin Change, Longest Palindrome, Knapsack

### Divide & Conquer
Used in: Sorting, Searching, Tree Problems
Problems: Merge Sort, QuickSort, Invert Tree

## Interview Day Checklist

### Before Interview
- [ ] Clarify requirements and constraints
- [ ] Discuss tradeoffs
- [ ] Draw diagrams (architecture, class, sequence)
- [ ] Explain your approach before coding
- [ ] Ask about scale/requirements
- [ ] Test with examples

### During Interview
- [ ] Listen fully without interrupting
- [ ] Ask clarifying questions
- [ ] Think out loud (show your process)
- [ ] Discuss complexity (time & space)
- [ ] Consider edge cases
- [ ] Optimize together with interviewer

### After Interview
- [ ] Thank the interviewer
- [ ] Ask about next steps
- [ ] Send thank you email
- [ ] Review and learn from it
- [ ] Keep improving

## Troubleshooting

### Port 3000 already in use?
```bash
lsof -i :3000  # Find what's using it
npm start -- --port 3001  # Use different port
```

### Content not loading?
- Check network tab in DevTools
- Ensure all files are in src/ subdirectories
- Check browser console for errors

### Progress not saving?
- Check if data/ directory exists (auto-created)
- Clear browser cache
- Check browser's local storage settings

### Need more content?
- Add markdown files to topic directories
- Service auto-loads them
- Restart server to refresh
